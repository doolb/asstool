字體大小

\fs<size>

\fs<10>
*<size>指的是文字縱向的像素數量。
} pos {

字幕位置

\pos(<X>,<Y>)

\pos(<470>,<260>)
設置本行字幕的位置
} move {

移動

\move(<x1>,<y1>,<x2>,<y2>)
\move(<x1>,<y1>,<x2>,<y2>,<t1>,<t2>)

\move(100,150,300,350)
當這行文字出現在屏幕上時，字幕位於（100，150），然後字幕以穩定的速度移動到（300，350） 接著消失。

\move(100,150,300,350,500,1500)
當這行文字出現在屏幕上時，字幕位於（100，150），在半秒後(500 毫秒)，字幕會在一秒內移動到（300，350）。

決定是否傾斜 字體。
1表示傾斜，0表示不傾斜。
} b {

粗體

\b1
\b0
\b<weight>

*大部分字體不支持自定義字體粗細；如支持，更改的粗細應為100的倍數；
*100最細，400正常，700為粗體，900為極粗。
e.g. {\b100}How {\b300}bold {\b500}can {\b700}you {\b900}get?
} u {

} bord {

邊框寬度

\bord<size>

*<size>可設為0，也可為小數；

    e.g. \bord5, \bord0.5, \bord0【無邊框】
    *<size>可設為0，也可為小數；
} xbord {

單獨設定橫向的字體邊框寬度(x)

縱向的字體邊框寬度(y)
*一行代碼中，在\xbord 或\ybord後使用\bord 指令的話，\xbord和\ybord效果會被覆蓋。
} shad {

陰影深度

\shad<depth>

【陰影作用於文字背後，出現在右下角；經過顏色設置可造成 3D字體假象】
        *<depth>可以為小數，不可為負數。
} xshad {

橫向的陰影深度(x)

\xshad<depth>

*<size>可設為0，可為小數，也可為負數。
} yshad {

縱向的陰影深度(y)

\yshad<depth>

*<size>可設為0，可為小數，也可為負數。
} be {

模糊邊緣

\be0
\be1
\be<strength>

\be1【模糊邊緣】\be0【不模糊邊緣】
*字體有邊框時以邊框顏色為準模糊邊緣，無邊框時以文字顏色為準模糊文字邊緣；
*模糊效果在字體較小時比較明顯，但是使用可以使文字看過去更柔和。

\be<strength>
*字體有邊框時以邊框顏色為準模糊邊緣，無邊框時以文字顏色為準模糊文字邊緣；
*設置值必須為整數；
*因為模糊效果為漸變色，設置值過高沒有意義（末端幾乎透明）。
} blur {

模糊邊緣

\blur<strength>

*字體有邊框時以邊框顏色為準模糊邊緣，無邊框時以文字顏色為準模糊文字邊緣；
    *設置值可為小數；
    *用法與\be<strength>幾乎一樣，但是因為算法不同，在設置值較高的時候有很好的表現，但要注意設置值過高會給CPU帶來壓力。

} fr {

文字旋轉

\fr<amount>

*\fr默認繞Z 軸（垂直於屏幕的軸）。
\fr (= \frz)<amount> 【繞Z軸旋轉（逆時針為正，順時針為負）】

e.g.

\frx<amount>

\fry<amount>

【繞 Y軸（豎直軸）旋轉(為正值時字左大右小，為負時字左小右大)】
} frz {

文字繞 Z軸（水平軸）旋轉

\frz<amount>

*\frz繞Z 軸（垂直於屏幕的軸）。
\fr (= \frz)<amount> 【繞Z軸旋轉（逆時針為正，順時針為負）】

e.g.

\fax<factor>

} fay {

文本在Y轴上倾斜

\fay<factor>

*一般而言<factor>取較小的值，小於2的值效果都非常明顯；
    *在旋轉命令之後，傾斜所應用的軸為旋轉之後的軸，與初始軸無關。
} fe {

設置透明度

\alpha&H<aa>
\1a&H<aa>
\2a&H<aa>
\3a&H<aa>
\4a&H<aa>

\alpha&H<aa> 【一次性設置以下四者的透明度】
        \1a&H<aa> 【文字原色透明度】
        \2a&H<aa> 【卡拉 ok效果中的第二顏色透明度】
        \3a&H<aa> 【字體邊框透明度】
        \4a&H<aa> 【字體陰影透明度】
        *經實踐Set alpha代碼內的&和H均可省略不寫；
        *<aa>採用十六進制（00【不透明】, 37【25%透明】80【50%透明】, FF【100%透明】）；
        *<aa>不區分大小寫；
        *超過80效果就不明顯了（也許是我打開方式不對_(:3」∠) _）。
} an {

柵格定位

\an<pos>

*\pos和\move會覆蓋\an指令效果，但是\an的特徵會保留；
（如\pos(300,500)\an1，字幕是居左的，在字幕有兩行以上的情況下每行開頭均居左，否則是居中）
  1. 底部 最左側
  2. 底部 居中
  3. 底部 最右側
  4. 中部 最左側
  5. 中部 居中
  6. 中部 最右側
  7. 頂部 最左側
  8. 頂部 居中
  9. 頂部 最右側
} k {

卡拉OK特效

\k<duration>
\K<duration>
\kf<duration>
\ko<duration>

單位為厘秒 (cs) ，100cs = 1000ms = 1s。
    \k<duration> 【卡拉 ok跳色效果，一個字一個字間斷變色】
    *<duration>的時間經過之後執行跳色命令。
    e.g. {\k50}{\k80}Think {\k25}of {\k60}me
    【50cs之後 "Think"從次要顏色跳為主要顏色， 80cs後"of"跳為原色，25cs後"me"跳為原色】

    \kf<duration> or \K<duration>
    【卡拉 ok填充效果（與\k效果相似，但是有從左到右填充效果，較為平滑） 】     
*在<duration>時間段內填充完色。     
e.g. {\kf80}Think {\kf25}of {\kf60}me
    【80cs內 "Think"從次要顏色由左至右變成主要顏色， 25cs內"of"漸變為原色，60cs內"me"漸變為原色】

    \ko<duration> 【邊框卡拉 ok跳色效果（與\k效果相似）】
    e.g. {\ko50}{\ko80}Think {\ko25}of {\ko60}me
    【一開始所有邊框都沒有，50cs之後 "Think"的邊框跳色， 80cs後"of"的邊框跳色，25cs後"me"的邊框跳色】
} q {

換行方式

\q<style>

0：智能換行，盡量使每行寬度一致，不能保持等寬時上行較寬。只有\N指令強制換行有效；
        1：於行尾詞處換行，僅\N斷行有效；
        2：不自動換行，\n及\N斷行有效；
        3：智能換行，盡量使每行寬度一致，不能保持等寬時下行較寬。只有\N指令強制換行有效。
} r {

重置字幕樣式

\r
\r<style>

e.g. -Hey\N{\rAlternate}-Huh?\N{\r}-Who are you?

【假設此條字幕樣式為"Default"，則"Hey" 為Default style，"Huh?" 被改成了"Alternate"，"Who are you?" 又被恢復為Default style】

} org {

旋轉原點

\org(<X>,<Y>)

*一旦旋轉/縮放中心被設定，則它就是固定不動的；
        *將該點設於極遠處、配合快速特效可以造成跳躍的視覺效果；(反正我不會用OTZ)
        *每條文本最多只能有一個org命令。
} fad {

淡入淡出

\fad(<fadein>,<fadeout>)

【開始時間到t1淡入，t2到結束時間淡出】
        *t1, t2單位為毫秒；
        *t1+t2必須小於最多等於本條字幕的持續時間；
        *t1=0，無淡入； t2=0，無淡出。
} fade {

淡入淡出(高級)

\fade(<a1>,<a2>,<a3>,<t1>,<t2>,<t3>,<t4>)

e.g. \fade (255, 32, 224, 0, 500, 2000, 2200)
【字幕從開始0s到0.5s由透明漸變為幾乎不透明，維持32的透明度1.5s，此後在0.2s之內由透明度32變為224（幾乎透明），此後維持幾乎透明狀態直至本條字幕時間結束】
*透明度值為十進制，從0（不透明）到255（完全透明）;
    *時間以毫秒計；時間為累計時間；
    *七個參數都必須有；
    *fade命令中透明度標記影響 1c-4c所有顏色。
    【t1之前透明度為a1，
    在t1到t2時間段透明度由a1漸變為a2，
    在t2到t3時間內透明度保持為a2，
    在t3到t4時間段透明度由a2漸變為a3，
    t4之後透明度保持為a3】
} t {

動態變換

\t(<style modifiers>)
\t(<accel>,<style modifiers>)
\t(<t1>,<t2>,<style modifiers>)
\t(<t1>,<t2>,<accel>,<style modifiers>)

e.g.
        {\1c&HFF0000&\t(\1c&H0000FF&)}Hello!
        【“Hello!”由藍色變為紅色】

        {\an5\t(0,5000,\frz3600)}Wheee
        【“Wheee”在5s內勻速逆時針轉10圈】

        {\an5\t(0,5000,0.5,\frz3600)}Wheee
        【“Wheee”在5s內減速逆時針轉10圈】

        {\an5\fscx0\fscy0\t(0,500,\fscx100\fscy100)}Boo!
        【Boo!在0.5s內由一個點變成文字】
\t(<style modifiers>)
        【在字幕持續時間內由\t前的狀態變成( )內狀態】

        \t(<accel>, <style modifiers>)
        【同上，速度非線性】

        \t(<t1>, <t2>, <style modifiers>)
        【在t1~t2時間內由\t前的狀態變成( )內狀態】

        \t(<t1>, <t2>, <accel>, <style modifiers>)
        【同上，速度非線性】

*只有以下tags可以用於變換：
字體 幾何 其他效果
\fs \fscx \bord
\fsp \fscy \xbord
\c \frx \ybord
\1c \fry \shad
\2c \frz \xshad
\3c \fr \yshad
\4c \fax \clip
\alpha \fay \iclip
\1a   \be
\2a   \blur
\3a    
\4a    
*t1和t2單位為毫秒，時間為累計時間；
        *t1~t2的時間間隔為特效發生的時間，如果未指定，則為整條字幕持續時間；
        *<accel>可以實現非線性速度變化，具體為：
        <accel> 等於 1，速度為線性；
        <accel>在0和1之間 ，開始速度快結束速度慢；
        <accel> 大於 1，開始速度慢結束速度快。
        *自己實踐的時候發現\(i)clip不好用，不如說和其代碼本身說明是矛盾的，無解ING
} clip {

矩形蒙板 (rectangle)

\clip(<x1>,<y1>,<x2>,<y2>)
\iclip(<x1>,<y1>,<x2>,<y2>)

e.g.
\clip (0, 0, 704, 245)
【只有矩形內的內容被顯示】
        *默認為矩形，以 (x1, y1), (x2, y2) 分別為對角線端點坐標。

圖形蒙板 (vector drawing)

\clip(<drawing commands>)
\clip(<scale>,<drawing commands>)
\iclip(<drawing commands>)
\iclip(<scale>,<drawing commands>)

e.g. 
\clip(1,m 50 0 b 100 0 100 100 50 100 b 0 100 0 0 50 0)
*代碼使用方法、效果與Clip (rectangle) 一致，請結合Drawing tags自己探索
} iclip {

矩形蒙板 (rectangle)

\iclip(<x1>,<y1>,<x2>,<y2>)

e.g.
\iclip (0, 0, 704, 245)
【只有矩形內的內容不被顯示】
        *默認為矩形，以 (x1, y1), (x2, y2) 分別為對角線端點坐標。
        *iclip的影響效果是整行、無視代碼放置位置的。

圖形蒙板 (vector drawing)

\iclip(<drawing commands>)
\iclip(<scale>,<drawing commands>)

e.g. 
\iclip(1,m 50 0 b 100 0 100 100 50 100 b 0 100 0 0 50 0)
*代碼使用方法、效果與Clip (rectangle) 一致，請結合Drawing tags自己探索
} p {

繪圖代碼

\p<0/1/..> - Toggle drawing mode

\p1 【激活繪圖命令，1：1繪圖】
    \p0 【關閉繪圖】
    \p2 【激活繪圖命令，用2倍的分辨率繪圖，2：1繪圖】
    \p4 【激活繪圖命令，用大8倍 (2^ (4-1)) 的分辨率繪圖，8：1繪圖】
    * n > 0，n為所擴大的範圍，用2^（n-1）轉化（可使所做圖像更精細）。
} pbo {

基線偏移

\pbo - Baseline offset

e.g. \pbo-50【圖像沿Y軸向上平移50像素】
\pbo100【圖像沿Y軸向下平移100像素】
【圖像沿y軸平移】
*該命令只應用於y軸（縱向）。
